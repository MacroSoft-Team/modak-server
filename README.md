# Modak Server

## Git Convention 가이드

### **Commit Types**

| Type    | Description                          |
|---------|--------------------------------------|
| **feat**  | 기능 or UI 수정                      |
| **fix**   | 버그 수정                              |
| **ref**   | 리팩토링                          |
| **style** | 코드 형식, 주석, or 정렬 정리|
| **chore** | 패키지 or 프로젝트 파일 설정  |
| **docs**  | 문서                        |
| **test**  | 테스트                              |

---

### **Issue 네이밍 형식**
- `[타입/#이슈번호] 설명`  
  > **[FEAT/#1]** 기능 추가

---

### **Branch 네이밍 형식**
- `타입/#이슈번호/작업명`  
  > **feat/#1/기능 추가**

---

### **Commit Message 형식**
- `타입: #이슈번호 작업내용 설명`  
  > **feat: #1** 기능 추가

---

### **Pull Request (PR) 형식**
- `[타입/#이슈번호] 설명`  
  > **[FEAT/#1]** 기능 추가
